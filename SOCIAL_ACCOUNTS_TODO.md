# Social Accounts TODO

Use this checklist to set up and launch `runfabric` community channels.

## 1. X / Twitter

- Reserve handle (`@runfabric`, fallback variants).
- Upload brand avatar, header, and bio with project link.
- Pin launch tweet with quickstart and docs links.
- Configure 2FA and assign at least two maintainers as admins.

## 2. Community Slack

- Create workspace and core channels (`#announcements`, `#help`, `#contributors`, `#showcase`).
- Add moderation guidelines and code-of-conduct link.
- Configure invite flow and anti-spam defaults.
- Add welcome message linking `docs/QUICKSTART.md` and `CONTRIBUTING.md`.

## 3. Serverless Meetups

- Define meetup format (monthly virtual, quarterly hybrid).
- Create organizer profile and event page.
- Prepare first agenda: intro, provider demo, roadmap.
- Publish RSVP links on README and social channels.

## 4. Stack Overflow

- Decide canonical tag strategy (`runfabric`, possibly `runfabric-framework`).
- Publish first Q&A seed posts (install, credentials, multi-provider deploy).
- Add answer templates for common onboarding questions.
- Track unanswered questions weekly.

## 5. Facebook

- Create official page/group and claim branding.
- Add project description, website, and docs links.
- Define posting cadence (release notes, examples, meetup reminders).
- Assign moderator roles and response SLA.

## 6. Contact Us

- Create support inbox (for example `contact@...`).
- Add response policy (triage time, support scope).
- Publish contact options in README and website.
- Add security reporting path separate from general contact.

## 7. Cross-channel Launch Tasks

- Align branding assets and bio text across all platforms.
- Publish a single launch announcement linking all channels.
- Track baseline metrics (followers, joins, engagement, support response time).
