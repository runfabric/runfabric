# runfabric `0.1.0-beta.0`

## Highlights

- Added state operations command group:
  - `runfabric state pull|list|backup|restore|force-unlock|migrate|reconcile`
- Added local development workflow:
  - `runfabric call-local --serve --watch`
  - `runfabric dev --preset <http|queue|storage>`
- Added traces/metrics command support:
  - local artifact-derived observability by default
  - provider-native override commands via `RUNFABRIC_*_TRACES_CMD` and `RUNFABRIC_*_METRICS_CMD`
- Added migration bootstrap command:
  - `runfabric migrate --input ./serverless.yml --output ./runfabric.yml`
- Added Node-first handler wrappers:
  - `createHandler` in `@runfabric/runtime-node` (Express/Fastify/NestJS auto-detection)
- Added typed trigger/schema expansion:
  - `http`, `cron`, `queue`, `storage`, `eventbridge`, `pubsub`, `kafka`, `rabbitmq`
- Added release/compatibility checks in CI:
  - schema compatibility checks against official examples
  - provider contract checks

## Notes

- Current production-ready runtime path remains `runtime: nodejs`.
- Remote state backends continue to run in simulated local-storage mode for dev/test in this beta.
