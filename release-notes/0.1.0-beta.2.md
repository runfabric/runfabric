# runfabric `0.1.0-beta.2`

## Highlights

- Improved release tooling:
  - added release note scaffold command: `pnpm run release:notes:create -- --version <version>`
  - publish flow now wires `NPM_TOKEN` into a temporary npm user config during `release:publish`, so local `.npmrc` token setup is not required for the script
- Aligned package metadata for this beta:
  - bumped all publishable packages to `0.1.0-beta.2`
  - updated repository URLs to `https://github.com/runfabric/runfabric.git`
  - updated CLI provider peer dependency ranges to `^0.1.0-beta.2`
- Added package READMEs for npm package pages:
  - `@runfabric/core`, `@runfabric/planner`, `@runfabric/builder`, `@runfabric/runtime-node`
  - all `@runfabric/provider-*` packages
- Updated product-positioning docs language:
  - README/comparison wording now explicitly frames managed-serverless auto-scaling and low idle-cost overhead while keeping runfabric's portability-first scope

## Breaking Changes

- None.

## Notes

- Publish scope is `@runfabric/*`; npm org access and token permissions must match that scope.
- Sign and verify this file before publish:
  - `pnpm run release:notes:sign -- --version 0.1.0-beta.2`
  - `pnpm run release:notes:verify -- --version 0.1.0-beta.2`
