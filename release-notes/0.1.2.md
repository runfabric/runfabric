# runfabric `0.1.2`

## Highlights

- Enforced code health guardrails across the monorepo:
  - `max-lines=600`
  - `max-lines-per-function=50`
  - `complexity=20`
  - `max-top-level-functions=5`
  - `max-classes-per-file=1`
  - `max-type-declarations=1` (type/interface/enum)
- Refactored oversized modules into focused files to improve maintainability and lint compliance:
  - split CLI command implementations (`call-local`, `deploy`, `docs`, `init`, `migrate`)
  - split planner config parsing into dedicated modules (`yaml`, `triggers`, `state-config`, `extensions`, `stages`, shared readers/helpers)
  - split state backend implementation into dedicated modules (`record-utils`, `config-utils`, `file-backend`, `keyvalue-backend`, `backup`)
  - split AWS Lambda provider internals into dedicated deploy and metadata modules
- Added non-blocking common-utils detection in syntax checks to highlight repeated helper functions across files for future consolidation.

## Breaking Changes

- None.

## Notes

- New structural guardrails use a progressive rollout baseline for legacy files that still need decomposition.
- Existing release and compatibility gates remain green with this release:
  - `check:syntax`
  - `check:compatibility`
  - `release:check`
